## 1. The OneLake Concept (The "OneDrive for Data")
* **Unified Data Lake:** OneLake is the logical, centralized data lake for the entire tenant, built on Azure Data Lake Storage (ADLS) Gen2. 
* **One Copy Principle:** Data is stored once in an open format (Delta Lake) and accessed by multiple engines (SQL, Spark, Power BI) without movement or duplication.
* **Shortcuts:** Allows virtualizing data from other sources (ADLS, AWS S3, Google Cloud Storage) into OneLake without moving the underlying files.

## 2. Capacity & Licensing (Enterprise Scaling)
* **Capacity SKUs:**
    * **F-SKUs (Fabric):** Azure-based, pay-as-you-go capacity. Managed via the Azure Portal.
    * **P-SKUs (Power BI Premium):** Monthly/yearly commitment. Fabric features are enabled on existing Power BI Premium capacities.
* **The F64 Threshold:** For users with a "Free" license to view Power BI content in Fabric, the workspace must be backed by an **F64 or larger** SKU. On smaller SKUs, every viewer requires a Pro or PPU license.
* **Capacity Units (CU):** A measure of compute power. Unlike Power BI's fixed v-cores, Fabric uses a shared pool of CUs that can be smoothed over time to handle bursts in workload.

## 3. OneLake Security & Governance
* **Authentication:** Strictly relies on Entra ID. Access is managed through Workspace Roles and Item Permissions.
* **OneLake Data Access Roles (Preview):** Defines granular security at the folder and table level. Roles consist of:
    * **Data:** Specific tables or folders.
    * **Permissions:** Read/Write access.
    * **Members:** Users or Entra ID security groups.
* **Unified Governance:** Integration with Microsoft Purview ensures that sensitivity labels and DLP policies are inherited from the source and enforced across all Fabric workloads.

## 4. Microsoft 365 & Copilot Integration
* **Direct Lake Mode:** A Power BI feature that reads Delta tables directly from OneLake without the need to import data or use slow DirectQuery.
* **Excel & Teams:** Fabric items (like Semantic Models) can be explored natively in Excel or shared via Teams, maintaining the original security context.
* **Copilot for Fabric:** Assists in generating DAX queries, Python code in notebooks, and data pipelines. It respects the tenant's security boundaries and does not use organizational data to train public models.

## 5. Data Loss Prevention (DLP) & Information Protection
* **DLP for Fabric:** Compliance teams can create policies in Purview to detect sensitive data (e.g., Credit Card numbers) in Lakehouses and Warehouses, automatically restricting access when a violation is found.
* **Sensitivity Labels:** Labels applied in Fabric (e.g., "Highly Confidential") are visible in the OneLake Catalog and travel with the data if it is exported to Power BI or Excel.
* **Public APIs:** Sensitivity labels can be programmatically managed via Fabric APIs to support large-scale automated classification.

## 6. Essential PowerShell Cmdlets (Az.Fabric & PowerBIMgmt)
* **Connection:** `Connect-AzAccount` or `Connect-PowerBIServiceAccount`
* **Capacity Management (Az.Fabric Module):**
    * `Get-AzFabricCapacity` (Lists all Fabric capacities in the subscription).
    * `Update-AzFabricCapacity -Name "MyCapacity" -Sku "F64"` (Scales capacity up or down).
    * `Suspend-AzFabricCapacity` / `Resume-AzFabricCapacity` (Pauses billing for pay-as-you-go SKUs).
* **Workspace Management (PowerBIMgmt):**
    * `Get-PowerBIWorkspace -Scope Organization` (Lists workspaces, including Fabric-enabled ones).
    * `Set-PowerBIWorkspace -Id <ID> -CapacityId <CapacityID>` (Assigns a workspace to a specific Fabric capacity).
